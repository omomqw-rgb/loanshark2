# Global CSS Rules Classification

| 규칙 ID | 분류 | 원래 의도 | 지금 구조에서 문제점 |
|---|---|---|---|
| 1 | 리셋 목적 | 전역 바탕/기본 글자색 지정 | 문제는 없으나, 화면별 CSS 범위 분리 관점에서는 '의도된 전역'임을 명확히 해야 함 |
| 2 | 리셋 목적 | 전역 배경/기본 글자색 지정(테마) | 문제는 없으나, 다른 파일의 전역 색 규칙과 충돌 시 기준점이 됨 |
| 3 | 리셋 목적 | 전역 배경/기본 글자색 지정(중복) | 동일/유사 선언 중복으로 cascade 추적 난이도 증가 |
| 4 | 특정 화면 전용인데 전역으로 퍼진 경우 | 캘린더 요일/주말 색상 보정(토/일 강조) | root 누락(또는 selector 혼재)로 타 화면의 동일 클래스가 생기면 색상 오염 가능. 현재도 전역 우선순위로 다른 색을 덮을 위험 |
| 5 | 특정 화면 전용인데 전역으로 퍼진 경우 | 캘린더 요일/주말 색상 보정(토/일 강조) | root 누락(또는 selector 혼재)로 타 화면의 동일 클래스가 생기면 색상 오염 가능. 현재도 전역 우선순위로 다른 색을 덮을 위험 |
| 6 | 특정 화면 전용인데 전역으로 퍼진 경우 | 캘린더 요일/주말 색상 보정(토/일 강조) | root 누락(또는 selector 혼재)로 타 화면의 동일 클래스가 생기면 색상 오염 가능. 현재도 전역 우선순위로 다른 색을 덮을 위험 |
| 7 | 특정 화면 전용인데 전역으로 퍼진 경우 | 캘린더 요일/주말 색상 보정(토/일 강조) | root 누락(또는 selector 혼재)로 타 화면의 동일 클래스가 생기면 색상 오염 가능. 현재도 전역 우선순위로 다른 색을 덮을 위험 |
| 8 | 특정 화면 깨짐 방지용 | 카드 액션 영역의 plain 버튼 가독성 확보 | 전역 클래스 재사용 시 다른 화면의 `.btn-plain` 색을 강제할 수 있어 scope 분리 필요(컴포넌트 범위로 명확화) |
| 9 | 브라우저 기본 스타일 제거 | 버튼 hover 시 기본적인 인터랙션 배경 제공 | 전역 적용 자체는 의도일 수 있으나, 화면별 hover 컬러가 필요할 때 충돌 가능 |
| 10 | 잘못된 범용 적용 | 배지 텍스트 색을 상위 요소와 동기화하려는 시도(추정) | 이미 badge-loan/claim 색 정의를 무력화(`color: inherit !important`)하여 타입 색 표현을 제거 |
| 11 | 브라우저 기본 스타일 제거 | placeholder 기본 색을 통일 | 전역 적용 자체는 의도일 수 있으나, 특정 화면에서 placeholder 색을 토큰으로 제어할 때 충돌 가능 |
| 12 | 브라우저 기본 스타일 제거 | 포커스 링/보더 색을 통일 | 전역 포커스 스타일이 특정 화면/모달의 강조색과 충돌할 수 있음 |
| 13 | 브라우저 기본 스타일 제거 | 입력 UI 기본 배경/테두리/텍스트 색을 통일 | 전역 입력 스타일이 모달/특정 화면의 디자인 토큰을 덮을 수 있음(필요 시 컴포넌트 단위로 레이어 분리 필요) |
| 14 | 특정 화면 전용인데 전역으로 퍼진 경우 | 채무자 상세 헤더 버튼 스타일(배경/색) | root 미부여로 동일 클래스가 다른 화면에서 쓰일 경우 버튼 색 오염 가능 |
| 15 | 특정 화면 전용인데 전역으로 퍼진 경우 | 동일 규칙 중복(정리 필요) | 중복으로 cascade 추적 난이도 증가 |
| 16 | 특정 화면 전용인데 전역으로 퍼진 경우 | 채무자 헤더 아이콘 버튼 색 지정 | 전역 `.debtor-header-*` 클래스가 다른 화면에서 재사용될 경우 색 강제 |
| 17 | 특정 화면 전용인데 전역으로 퍼진 경우 | 채무자 패널 내부 텍스트/상태 색 동기화 | scope 누락 시 다른 화면의 유사 클래스/컴포넌트를 오염 |
| 18 | 특정 화면 전용인데 전역으로 퍼진 경우 | 검색 인풋 배경/글자색 | 전역 인풋 스타일과 충돌 시 cascade가 복잡해지고 다른 화면에도 영향 가능 |
| 19 | 특정 화면 전용인데 전역으로 퍼진 경우 | 폼 액션 버튼(저장/취소 등) 배경/테두리 | 전역 `.btn-inline` 재사용 시 색이 강제됨 |
| 20 | 특정 화면 전용인데 전역으로 퍼진 경우 | 동일 규칙 중복(정리 필요) | 중복으로 cascade 추적 난이도 증가 |
| 21 | 특정 화면 전용인데 전역으로 퍼진 경우 | 주/일 상세 아이템의 기본 금액 색 지정(캘린더 UI) | `debtors.css`에서 캘린더 UI 클래스에 접근해 다른 화면까지 오염 가능 |
| 22 | 잘못된 범용 적용 | 자식 요소 색을 상속시키려는 리셋(추정) | `color: inherit !important`로 상태/타입 색 규칙을 무력화하여 캘린더 색 표현을 죽임 |
| 23 | 특정 화면 전용인데 전역으로 퍼진 경우 | 연체 상태 금액 색 지정(캘린더 UI) | 캘린더 전용 규칙이 debtors로 퍼져 scope 경계 붕괴 |
| 24 | 특정 화면 전용인데 전역으로 퍼진 경우 | 완납 상태 금액 색 지정(캘린더 UI) | 동일 |
| 25 | 특정 화면 전용인데 전역으로 퍼진 경우 | 부분납 상태 금액 색 지정(캘린더 UI) | 동일 |
| 26 | 과거 레거시 보정 | 칩 컴포넌트의 텍스트/배경을 주변 테마에 맞춰 상속 | 전역 상속 규칙은 의도일 수 있으나, 특정 화면에서 칩 색을 독립 제어하려면 컴포넌트 레이어 분리 필요 |
| 27 | 특정 화면 전용인데 전역으로 퍼진 경우 | 카드/패널/사이드패널 표면(배경/보더/그림자) 통일 | modal.css에 정의돼있어 '모달 파일이 전역 표면을 지배'하는 구조. 화면/컴포넌트 경계가 흐려짐 |
| 28 | 특정 화면 전용인데 전역으로 퍼진 경우 | 모달 컨테이너 배경/테두리/텍스트 등 기본 스킨 | root 미부여 시 모달과 무관한 곳에 `.modal` 클래스가 생기면 오염 가능 |
